Hướng Dẫn Git Toàn Diện

1. Lệnh Git Cơ Bản & Nâng Cao

git init

Khởi tạo Git repository tại thư mục hiện tại.

git clone <url>

Sao chép repository từ GitHub về máy cục bộ.

git add .

Thêm tất cả thay đổi vào staged area.

git commit -m "message"

Commit các thay đổi với thông điệp.

git push origin main

Đẩy thay đổi lên nhánh main.

git pull

Tải các thay đổi mới nhất từ remote repository về máy của bạn.

git fetch

Lấy về các thay đổi từ remote mà không tự động gộp.

git merge

Gộp thay đổi từ một nhánh vào nhánh hiện tại.

git status

Kiểm tra trạng thái các tệp tin và nhánh hiện tại.

git branch

Liệt kê các nhánh hiện có hoặc tạo nhánh mới.

2. Git Flow (quy trình chuyên nghiệp)

  • main: chứa code ổn định, đã release
  • develop: nhánh chính cho phát triển
  • feature/ten-tinh-nang: dùng cho từng tính năng nhỏ
  • release/1.0.0: chuẩn bị bản phát hành
  • hotfix/ten-sua-loi: sửa lỗi gấp cho bản production

Ví dụ tạo nhánh feature và merge:

git checkout develop git checkout -b feature/tim-kiem ... code và commit ... git checkout develop git merge feature/tim-kiem

3. Hướng dẫn sử dụng GitHub giao diện web

  1. Vào https://github.com và đăng nhập
  2. Nhấn “New repository”
  3. Đặt tên repo và nhấn “Create repository”
  4. Làm theo hướng dẫn: thêm remote và push
  5. Trong mỗi repo, tab Code là nơi chứa source
  6. Tab Pull Requests: tạo yêu cầu gộp nhánh
  7. Tab Actions: thiết lập CI/CD với GitHub Actions

4. Câu hỏi thường gặp (FAQ)

Git và GitHub khác gì nhau?

Git là công cụ quản lý mã nguồn cục bộ. GitHub là nơi lưu trữ Git repo online, hỗ trợ làm việc nhóm.

Làm sao sửa commit cuối?

git commit --amend để chỉnh lại nội dung hoặc message commit cuối.

Cách xóa file khỏi Git mà không xóa trên máy?

git rm --cached tenfile

Làm sao cập nhật repo đã có từ GitHub về máy?

Dùng git pull origin main để đồng bộ code mới về local.
