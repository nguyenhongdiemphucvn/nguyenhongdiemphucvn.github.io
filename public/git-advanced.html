1. Quản lý nhiều remote

Khi bạn fork một dự án, bạn sẽ có 2 remote: origin và upstream.

git remote add upstream https://github.com/original-project.git
git fetch upstream
git merge upstream/main

2. Rebase tương tác

Giúp bạn làm sạch lịch sử commit trước khi gửi pull request.

git rebase -i HEAD~5

Bạn có thể pick, reword, squash hoặc drop commit.

3. Git Hooks

Tự động hóa kiểm tra trước khi commit hoặc push.

Ví dụ hook kiểm tra commit message:

#!/bin/sh
commit_msg=$(cat "$1")
if ! echo "$commit_msg" | grep -qE "^(feat|fix|docs|refactor): "; then
  echo "❌ Commit message phải bắt đầu bằng từ khóa hợp lệ!"
  exit 1
fi

Đặt vào .git/hooks/commit-msg và cấp quyền thực thi.

4. Git Worktree

Cho phép làm việc nhiều nhánh cùng lúc mà không cần clone nhiều lần.

git worktree add ../feature-a feature-a

5. Git Bisect

Tìm commit gây ra lỗi bằng cách chia đôi lịch sử.

git bisect start
git bisect bad
git bisect good v1.0.0
# Test từng bước và đánh dấu good/bad

6. GPG Commit

Ký commit để chứng minh danh tính (GitHub hiển thị Verified).

git config --global user.signingkey ABCD1234
git commit -S -m "feat: cập nhật bảo mật"

7. Git Submodule

Dùng khi dự án của bạn phụ thuộc vào dự án khác.

git submodule add https://github.com/example/lib-b.git libs/lib-b
git submodule update --init --recursive
